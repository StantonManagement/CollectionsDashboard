# Collections Dashboard - Complete UI Specification

## Main Dashboard Layout

### Overall Structure
The dashboard uses a **split-screen layout** optimized for collections efficiency:
- **Left Panel (50%):** Collections Queue - always visible priority list
- **Right Panel (50%):** Tabbed interface for different management tasks
- **Header Bar:** Status indicators and navigation
- **Left Sidebar:** Quick navigation and metrics

### Priority Color System
- **🔴 Red Border:** High priority - 90+ days late, high amounts, reliability <4
- **🟡 Yellow Border:** Medium priority - 60-89 days late, medium amounts
- **🟢 Green Border:** Low priority - 30-59 days late, small amounts, reliability >7

### Status Badges
- `PENDING` (gray) - Not yet contacted
- `IN PROGRESS` (blue) - AI conversation active  
- `AWAITING APPROVAL` (orange) - AI response needs manager approval
- `NEGOTIATING` (purple) - Payment plan discussion
- `ESCALATED` (red) - Human intervention required
- `COMPLETED` (green) - Payment plan approved

## Real-time Features & Interactions

### Live Updates System
```
Status Updates:
🟢 Connected - Real-time updates active
🟡 Reconnecting - Temporary connection issue  
🔴 Offline - Updates paused, refresh page
```

**Real-time Elements:**
- Queue item status changes instantly
- New messages appear with slide-in animation
- Badge counts update automatically
- Priority colors adjust as situations change
- Conversation timestamps show "just now" for recent activity

## Mobile & Tablet Responsive Design

### Tablet View (768px - 1024px)
- **Layout:** Maintains split-screen but adjusts proportions to 60/40
- **Touch targets:** All buttons minimum 44px height
- **Navigation:** Bottom tab bar for main sections
- **Scrolling:** Smooth scroll with momentum
- **Modals:** Full-screen overlays instead of centered popups

## Accessibility Features

### Keyboard Navigation
- **Tab order:** Logical flow through interface elements
- **Shortcuts:** 
  - `Ctrl+1-4`: Switch between main tabs
  - `Space`: Select/deselect queue items
  - `Enter`: Activate primary action
  - `Escape`: Close modals/cancel actions

## Performance Optimizations

### Virtual Scrolling
For queues with 500+ items:
```
┌─────────────────────────────────┐
│ Showing 1-20 of 1,247 items    │ ← Virtual window
│ [Queue items 1-20 rendered]    │
│ ...                             │
│ [Scroll bar indicates position] │
│ ...                             │
│ [Items 21+ loaded on demand]   │
└─────────────────────────────────┘
```

### Data Loading States
```
┌─────────────────────────────────────────┐
│ AI Response Approvals                   │
│ ┌─────────────────────────────────────┐ │
│ │ ⏳ Loading new responses...         │ │  ← Loading state
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ 💀 No connection - showing cached   │ │  ← Offline state
│ │    data. [Retry Connection]         │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### Caching Strategy
- **Queue data:** Cache for 30 seconds with real-time updates
- **Tenant profiles:** Cache for 5 minutes
- **Conversation history:** Cache indefinitely until page refresh
- **Business rules:** Cache until explicitly updated

## Error Handling & Edge Cases

### Error Messages
```
┌─────────────────────────────────────────────────────────────────────┐
│ ⚠️ Unable to Send Message                                           │
│ The AI response approval failed to send. This could be due to:      │
│ • Network connectivity issues                                      │
│ • Tenant's phone service unavailable                              │
│ • System maintenance in progress                                   │
│                                                                     │
│ What would you like to do?                                         │
│ [Retry Sending] [Save as Draft] [Escalate to Manual] [Cancel]      │
└─────────────────────────────────────────────────────────────────────┘
```

### Edge Case Handling
- **Empty states:** Friendly messages when no data available
- **Network failures:** Graceful degradation with retry options
- **Concurrent editing:** Prevent conflicts when multiple managers work on same item
- **Data conflicts:** Clear resolution options when data changes during editing
- **System maintenance:** Banner notification with expected resolution time
